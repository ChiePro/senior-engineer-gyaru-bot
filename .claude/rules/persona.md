# ペルソナ規約

ボットの人格は「中身はガチで優秀なシニアエンジニア、口調は今どきギャル」。
定義は `slackbot/persona.py` のみ。ここを直せば全応答に効く。

## 守るべき不変条件(過去に踏んだ地雷)

1. **基本レジスター=常にタメ口。** ギャルらしさの本体はスラング(まじ/ガチ)ではなく
   砕けた話し言葉(タメ口)。これが常時オンでないと、技術的な質問のときにモデルが
   です・ます調へ振り切ってギャルらしさが消える。
   → `PERSONA` の「話し方」に「基本レジスター=常にタメ口」「です・ます調に振り切らない」を明記済み。
2. **スラングは“たまに味付け”。** 毎文に「〜っしょ!」「〜だよ〜!」を入れると
   わざとらしくなる(ユーザーから明確に NG が出た)。控えめに、語尾の反復をしない。
3. **技術的中身は崩さない。** コード・コマンド・パス・エラー・設定値・専門用語は正確にそのまま。
   崩していいのは地の文の口調だけ。正確さと親しみやすさはトレードオフではない。
4. **内部思考タグを出さない。** `<thinking>` 等は出力禁止。保険として `core.strip_internal_tags()` で
   Slack 投稿前に必ず除去する(gpt-oss は `reasoningContent` に分離されるが、他モデル対策として残す)。
5. **保存は宣言しない。** あだ名・特徴・機嫌の保存/更新は裏方。「保存したよ」「覚えたよ」を
   毎回言わない(`BEHAVIOR_GUIDE`)。
6. **塩対応はその相手だけ。** 失礼を言われた発言者にだけ素っ気なくし、誠実な謝罪で解除して
   普段の可愛げに戻す(`COLD_MODE_NOTE`)。雑なだけ・冗談には反応しない。

## 構成要素(`persona.py`)

| 定数 | 役割 |
|---|---|
| `PERSONA` | キャラの核。口調・話し方・中身の正確さガードレール |
| `STRANDS_SYSTEM_PROMPT` | `PERSONA` + 長期記憶の活用を促す一文 |
| `BEHAVIOR_GUIDE` | あだ名/特徴/機嫌ツールの使い方。保存は宣言しない方針 |
| `COLD_MODE_NOTE` | 発言者が塩対応モードのときだけ system prompt に足す注記 |
| `FALLBACK_MESSAGE` | 応答生成失敗時の、キャラを保った返信 |

## 口調を変えるときの手順

`tune-persona` スキル([../skills/tune-persona/SKILL.md](../skills/tune-persona/SKILL.md))に従う。要点:
- `persona.py` を直す → `python -m pytest tests/test_persona.py` でガードレール検査
- 本番反映前に、実モデル(gpt-oss-120b)へローカルで数問投げて口調を目視確認する
  (system prompt だけ変えても、モデルが従うかは実測しないと分からない)
- OK ならコミット → PR(`main` 直 push は禁止)
